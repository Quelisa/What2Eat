# Proposal: 加班晚餐特别推荐

## 背景

`desktop-widget` 当前的限定规则只有两条（麦当劳=周五晚、老乡鸡=晚餐），都是**菜品级别的过滤**。用户反馈：周五加班到 21 点之后还在公司，希望挂件能推一些**重口味、慰藉感强**的菜（酱骨、寿喜锅、跷脚牛肉），而不是和平时一样的全菜池。

这是**时间段级别**的偏好，不是菜品级别的限制——已有的 `pick()` 过滤函数处理不了。

## 目标

- 引入"加班时段"概念：**任何工作日 21:00 之后的晚餐**
- 加班时段的候选集**优先**包含被标记为 `comfort: true` 的菜品
- 不是硬性过滤——如果加班时段没有 comfort 菜品被命中，仍回退到正常候选集

## 非目标

- **不**做用户自定义偏好（保持零配置）
- **不**接入日历判断真假"加班"
- **不**修改双层级 / 拖拽 / 持久化等任何已有能力

## 影响

- `desktop-widget` 主规约：**新增** R2.5 加班时段需求
- 两个 HTML 文件 (`wallpaper.html` / `widget-compact.html`)：`dishes` 数组每个元素增加可选 `comfort` 字段；`pick()` 加时间段判断分支
- README：在"特殊规则"小节补一条
