# desktop-widget — Delta

新增需求 R2.5。其他需求保持原样。

## 需求 R2.5（新增）：加班晚餐慰藉推荐

**周一至周五 21:00 之后的"加班晚餐"时段，挂件优先推荐被标记为 `comfort: true` 的慰藉系菜品。**

### 场景 R2.5.1：进入加班时段

- **当**系统时间为周一至周五的 21:00–23:59
- **且**用户没有手动改过餐次（餐次仍由 `autoDetect()` 设置为「晚餐」）
- **那么**`pick()` 的候选集**必须**首先**仅包含**满足 `dish.comfort === true` 的菜品

### 场景 R2.5.2：加班时段的候选回退

- **当**加班时段下，按 R2.2/R2.3 等已有规则过滤后，comfort 子集为空
- **那么**`pick()` **应**回退到完整的、按已有规则过滤后的候选集，不让用户看到空白

### 场景 R2.5.3：手动覆盖

- **当**用户处于加班时段，但**手动**点了星期 / 餐次切换控件
- **那么**挂件**应**立即按用户选择的 (星期, 餐次) 走标准 R1.3 逻辑，**不再**应用加班 comfort 偏好

### 场景 R2.5.4：comfort 标记

- 至少以下菜品**必须**带 `comfort: true`：酱骨、寿喜锅、跷脚牛肉、瓦罐汤
- 其余菜品**默认**不带该字段（视为非 comfort）
